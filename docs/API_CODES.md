# B站 UP主激励计划 API 错误码参考

> 来源：`POST mission/receive` 接口返回值 `code` / `message`

## 成功

| code | message | 说明 |
|------|---------|------|
| 0 | — | 领取成功。`cdkey_content` 可能为空（部分活动不提供兑换码） |

## 终态错误（无需重试）

返回以下 code 后不会继续重试，worker 直接停止：

| code | message | 说明 |
|------|---------|------|
| 202031 | 任务奖励已经领取 | 该任务本期奖励已领过，领取时间窗口内不会再变 |
| 202032 | 无资格领取奖励 | 不满足活动参与条件（如等级、活动过期等） |
| 75255 | 库存已经使用完 | 全服库存耗尽，不会再补货 |
| -101 | 账号未登录 | Cookie（SESSDATA）失效，需重新扫码 |
| 202101 | 当前账号行为异常，无法领奖 | 同一天同一账号频繁领取触发风控，不会立即恢复 |

检测到以上 code 时，调度器会标记该链接为已领取并自动清空槽位（-101 和 202101 除外）。

## 可重试错误（频率限制）

不是终态，等一会儿可能成功：

| code | message | 说明 |
|------|---------|------|
| -509 | 请求过于频繁，请稍后再试 | IP 级限流 |
| -702 | 请求频率过高，请稍后再试 | 接口级限流，领取时段高峰期频发 |
| -705 | 请求过于频繁，请稍后再试 | 接口级限流 |

当前策略：worker 等 `retryInterval`（默认 1s）后重试，直到 `maxRetry`（默认 30）次耗尽或截止时间到。

## 领取时间限制

| code | message | 说明 |
|------|---------|------|
| 202120 | 任务领奖每日领取时间 | 不在每日领取时段内（通常为整点前后 1 分钟窗口） |

仅在非窗口期触发，正常调度不受影响。如果出现说明 cron 配置有误。

## 异常场景

| 现象 | 原因 | 说明 |
|------|------|------|
| 响应为 HTML（`<!DOCTYPE`） | B站风控 / WAF 拦截 | 短时间内请求过多触发验证码，JSON 解析失败 |
| -3 | 参数错误 | WBI 签名计算有误或请求参数缺失 |

## 接口参考

| 接口 | 方法 | 用途 |
|------|------|------|
| `/x/web-interface/nav` | GET | 校验登录态 + 获取 WBI 密钥 |
| `/x/activity_components/mission/info` | GET | 查询任务奖励信息 |
| `/x/activity_components/mission/receive` | POST | 提交领取请求 |
